## v1.2 Requirements

Requirements for Autoplay Stability & Polish milestone.

### Orchestration

- [x] **ORCH-01**: When autoplay starts, orchestrator triggers image generation API within 5 seconds of state transition
- [x] **ORCH-02**: Each autoplay iteration completes the full cycle: generate → evaluate → refine
- [x] **ORCH-03**: Multi-phase flow progresses through configured phases without manual intervention
- [x] **ORCH-04**: No timeout errors during normal autoplay operation

### State Reflection

- [x] **STATE-01**: Generate button shows accurate state (idle/generating/evaluating/refining)
- [x] **STATE-02**: Activity Modal displays real-time progress of current operation
- [x] **STATE-03**: Iteration count updates as cycles complete

### UI Lock

- [x] **LOCK-01**: Dimension inputs become read-only during autoplay
- [x] **LOCK-02**: Manual generate button disabled during autoplay
- [x] **LOCK-03**: Smart Breakdown disabled during autoplay
- [x] **LOCK-04**: User can abort autoplay at any time

### Architecture

- [x] **ARCH-01**: Orchestrator effect chain documented with data flow diagram
- [x] **ARCH-02**: Critical callback wiring verified and tested
- [x] **ARCH-03**: PATTERNS.md updated with lessons learned

### Visual Polish

- [x] **UI-01**: Activity Modal typography matches main page
- [x] **UI-02**: Activity Modal spacing and layout matches main page patterns
- [x] **UI-03**: Activity Modal color scheme consistent with app theme
